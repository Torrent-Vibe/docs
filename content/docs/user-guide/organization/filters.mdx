# Filters

Filters in Torrent Vibe allow you to create sophisticated queries to find and organize your torrents based on multiple criteria. This guide covers creating, managing, and using filters effectively.

## Understanding Filters

### What are Filters?

Filters are saved search queries that help you:
- **Quick Access**: Instantly find torrents matching specific criteria
- **Complex Queries**: Combine multiple search conditions
- **Saved Searches**: Reuse frequently used search patterns
- **Dynamic Updates**: Filters update automatically as torrents change
- **Shareable**: Export and import filter configurations

### Filter Types

#### Basic Filters
- **Status Filters**: Downloading, completed, paused, etc.
- **Category Filters**: Filter by assigned categories
- **Tag Filters**: Filter by applied tags
- **Size Filters**: Filter by torrent size ranges

#### Advanced Filters
- **Date Filters**: Filter by addition date, completion date
- **Speed Filters**: Filter by download/upload speeds
- **Ratio Filters**: Filter by share ratio
- **Tracker Filters**: Filter by tracker URLs

### Filter Conditions

#### Text-Based Conditions
- **Name Contains**: Torrent name contains specific text
- **Name Starts With**: Torrent name starts with text
- **Name Ends With**: Torrent name ends with text
- **Exact Match**: Torrent name exactly matches text

#### Numeric Conditions
- **Size Greater Than**: Torrents larger than specified size
- **Size Less Than**: Torrents smaller than specified size
- **Size Between**: Torrents within size range
- **Progress Greater Than**: Completion percentage above threshold

#### Date Conditions
- **Added Before**: Torrents added before specific date
- **Added After**: Torrents added after specific date
- **Completed Before**: Torrents completed before date
- **Completed After**: Torrents completed after date

#### Status Conditions
- **Status Is**: Specific torrent status
- **Status Is Not**: Exclude specific status
- **Has Error**: Torrents with errors
- **Is Stalled**: Torrents with no progress

## Advanced Filter Logic

### Boolean Logic

#### AND Conditions
- **All conditions must be true**
- Example: `Category = Movies AND Size > 1GB AND Status = Completed`

#### OR Conditions
- **At least one condition must be true**
- Example: `Tag = HD OR Tag = 4K OR Tag = BluRay`

#### NOT Conditions
- **Condition must be false**
- Example: `NOT Category = Software AND NOT Status = Error`

#### Complex Combinations
- **Nested Logic**: Combine AND, OR, NOT in complex ways
- **Group Conditions**: Use parentheses for grouping
- **Priority Order**: AND has higher priority than OR

### Filter Groups

#### Creating Groups
1. **Add condition group** in filter builder
2. **Set group logic** (AND/OR)
3. **Add conditions** to group
4. **Nest groups** for complex logic

#### Group Examples
```
Group 1 (AND):
  - Category = Movies
  - Size > 2GB
  - Year > 2020

Group 2 (OR):
  - Tag = HD
  - Tag = 4K
  - Tag = BluRay

Final: (Group 1) AND (Group 2)
```
