Manage multiple qBittorrent server connections and configure path mappings for cross-platform file access.

![Server Settings](/screenshots/TorrentVibeScreenshot_ServersSettings.png)

## Server Management

**Primary Server**: Main qBittorrent server connection
- None selected: No primary server
- Add server: Configure new connection
- Edit server: Modify existing settings

**Server Actions**:
- Add: New qBittorrent server connection
- Edit: Modify server configuration
- Delete: Remove server connection
- Test: Verify server accessibility

## Connection Details

**Server Name**: Descriptive identifier
- Custom name for server identification

**Host**: Server IP address or hostname
- Local: 127.0.0.1 or localhost
- Remote: Public IP or domain name

**Port**: qBittorrent Web UI port
- Default: 8080
- HTTPS: Usually 443

**Username/Password**: Web UI credentials
- Same as qBittorrent Web UI login

## Path Mapping

**Server Path**: Path on qBittorrent server
- Example: `/downloads/movies/`
- Maps to local equivalent

**Local Path**: Corresponding local directory
- Example: `/Volumes/External/Movies/`
- Must be accessible to Torrent Vibe

**Mapping Management**:
- Add mapping: Create new path translation
- Edit mapping: Modify existing mapping
- Delete mapping: Remove mapping
- Enable/Disable: Toggle mapping active state
