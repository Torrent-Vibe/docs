Torrent Vibe offers extensive customization options through its comprehensive settings panel. Access settings through the gear icon in the main interface.

## Settings Overview

The settings panel is organized into the following categories:

### Core Settings
- **[Downloads](downloads/)**: Configure download paths, behaviors, and file management
- **[Speed](speed/)**: Set global and alternate speed limits
- **[Connection](connection/)**: Configure peer connections and network settings
- **[BitTorrent](bittorrent/)**: Protocol-specific options and peer discovery

### Advanced Settings
- **[Web UI](web-ui/)**: Remote access configuration
- **[API Tokens](api-tokens/)**: External service integrations
- **[Servers](servers/)**: Multi-server management and path mapping
- **[Advanced](advanced/)**: Expert-level configuration options

### Interface Settings
- **[Appearance](appearance/)**: Theme, language, and UI customization
- **[Preferences](preferences/)**: Polling settings and desktop integration

## Quick Access

Most settings can be accessed directly from the main interface:
- Click the gear icon (⚙️) in the top-right corner
- Use keyboard shortcut: `Cmd/Ctrl + ,`
- Right-click menus for context-specific options

## Configuration Tips

### Performance Optimization
- Adjust speed limits based on your internet connection
- Configure appropriate peer limits
- Enable relevant protocol features (DHT, PEX, LSD)
- Set up automatic management for large libraries

### Security Best Practices
- Use strong passwords for Web UI
- Enable HTTPS for remote access
- Configure appropriate firewall rules
- Keep API keys secure and rotated

### Backup & Recovery
- Export settings regularly
- Document custom configurations
- Test settings changes incrementally
- Keep backup of working configurations

## Troubleshooting Settings

If settings aren't applying:
1. Check for conflicting options
2. Restart the application
3. Verify file permissions
4. Check logs for errors

For connection issues:
1. Test basic connectivity
2. Verify server credentials
3. Check firewall settings
4. Review network configuration
